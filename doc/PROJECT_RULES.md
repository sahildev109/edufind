# EduFind Rules

Stack:
- Next.js 14 App Router
- TypeScript
- Tailwind CSS
- Prisma
- PostgreSQL
- Gemini AI

Coding Rules:
- Use Server Components by default
- Use Client Components only when needed
- Use Prisma for all database access
- Use Zod validation
- Use Shadcn UI
- Use Zustand only for compare state and auth state
- Use TanStack Query for client fetching

Architecture:
- Follow EduFind Architecture Document
