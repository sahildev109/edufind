EduFind is a production-grade college discovery and decision-making platform that helps prospective students in India research, compare, and choose institutions with confidence. The platform differentiates itself through a powerful AI counselor (Gemini 2.5 Flash) that provides personalized college shortlisting and counseling based on a student's academic profile, budget, and goals — going far beyond simple search filters.
The system is architected as a full-stack Next.js 14 application using the App Router, with server-side rendering for public pages, React Server Components for data-heavy views, and client components only where interactivity is needed. Data is stored in a PostgreSQL database on Neon and accessed via Prisma ORM. Authentication uses JWT with refresh token rotation.

4.1.4 Implementation Details
Cursor pagination over offset: Offset pagination degrades at large offsets. Cursor-based (using the last item's cuid) stays O(1) regardless of how far into results the user is.
TanStack Query + useInfiniteQuery: Client fetches with infinite scroll. When the user scrolls within 200px of the bottom, the next page is automatically fetched and appended.
URL sync: nuqs library keeps filter state in the URL query string, enabling sharing and browser navigation without any client-side router complexity.
Debounced search: 300ms debounce on the text input prevents API spam on every keystroke.
Skeleton loading: While TanStack Query fetches, 12 CollegeCard skeletons are shown — no layout shift.

4.2.1 What It Does

4.2.2 Page Sections
4.2.3 ISR Strategy
The page is statically generated at build time for the top 100 colleges by NIRF rank (via generateStaticParams). For any college outside that set, it is generated on first request and then cached. The revalidate = 60 config means Vercel will regenerate in the background every 60 seconds if the page has been visited — users always see a fast cached page with at most 60-second-old data.
4.2.4 Save Button
The save/unsave button is an optimistic UI pattern: the button flips state immediately on click, makes the API call in the background, and reverts on failure. This gives instant feedback without waiting for a round-trip. The button shows a login modal for unauthenticated users instead of making the API call.

4.4.1 Auth Architecture

5.1 What It Does
The AI Counselor is a chat interface at /counselor. The student fills a profile form and then chats with the AI. The AI has access to the real college database through a structured context injection — it knows actual cutoffs, fees, and placement stats, not hallucinated data.
Core Capabilities
Profile-based shortlisting: Student inputs JEE Main/Advanced rank (or NEET/MHT-CET/other), preferred state, budget, and branch. Gemini ranks colleges from the database that match the profile.
Cutoff safety analysis: For each recommended college, the AI explains whether the student's rank is safe, borderline, or risky based on historical cutoff data.
Tradeoff explanations: AI explains pros/cons — e.g., "NIT Trichy has better placements but higher fees and is far from your home state."
Follow-up Q&A: Full multi-turn conversation context — student can ask "What about private colleges in Pune?" and the AI understands the prior context.
Comparison suggestions: AI can suggest adding specific colleges to the comparison tool via a structured action button embedded in its response.
5.2 Technical Implementation
5.2.1 RAG-lite Context Injection
Rather than full vector-database RAG, a lighter approach is used: before every AI request, the backend queries the database for colleges that roughly match the student's filters (state, budget range, branch) and injects the top 20 results as structured JSON into the Gemini prompt context. This grounds the AI on real data.

Current college discovery platforms force students to manually apply 8-10 filters, read dozens of pages, and mentally compare options themselves. The AI Counselor collapses this into a single conversation. This is the kind of feature that justifies a product's existence.

11.2 Security Measures
SQL injection: Prisma's parameterized queries prevent SQL injection by design.
XSS: React escapes all rendered strings. User-submitted review content is stripped of HTML server-side before storage.
CSRF: API routes check Content-Type: application/json — browser form submissions (potential CSRF vector) are rejected.
Rate limiting: All API routes: 100 req/min per IP. AI routes: 10 req/hour per user. Enforced via Upstash Redis sliding window.
Password hashing: bcrypt with cost factor 12. Passwords never stored in plaintext.
httpOnly cookies: Refresh token in httpOnly, Secure, SameSite=Strict cookie — inaccessible to JavaScript.

model College {
  id              String    @id @default(cuid())
  slug            String    @unique                 // URL-safe identifier
  name            String
  shortName       String?
  state           String
  city            String
  type            CollegeType  // GOVERNMENT | PRIVATE | DEEMED
  category        String       // IIT | NIT | IIIT | State | Pvt
  naacGrade       String?
  nirfRank        Int?
  establishedYear Int?
  description     String?      @db.Text
  imageUrl        String?
  websiteUrl      String?

  // Fees (annual, in INR)
  feesMin         Int
  feesMax         Int

  // Aggregated placement data (denormalized for fast listing)
  avgPackageLpa   Float?
  maxPackageLpa   Float?
  placementRate   Float?       // 0–100

  // Rating
  overallRating   Float        @default(0)
  reviewCount     Int          @default(0)

  // Relations
  courses         Course[]
  placements      PlacementStat[]
  reviews         Review[]
  savedBy         SavedItem[]
  facilities      CollegeFacility[]

  // Indexes
  @@index([state, city])
  @@index([type])
  @@index([nirfRank])
  @@index([feesMin, feesMax])
  @@index([overallRating])
  @@fulltext([name, city, state])  // PostgreSQL full-text search
}

Auth Flow:

1. User submits login → POST /api/auth/login
2. Server verifies password (bcrypt compare)
3. Issues: access_token (JWT, 15min) + refresh_token (random UUID, 7d)
4. refresh_token stored in DB + set as httpOnly cookie
5. access_token returned in response body → stored in memory (Zustand)

Token Refresh:
1. TanStack Query interceptor detects 401 response
2. Calls POST /api/auth/refresh with cookie (auto-sent by browser)
3. Server validates refresh token against DB
4. Issues new access_token + rotates refresh_token (old one deleted)
5. Original request retried with new access_token

Edge Middleware:
  - Runs at Vercel Edge before page render
  - Validates JWT signature (no DB call)
  - Redirects /dashboard → /login if token missing/invalid
5. LLM Feature — AI College Counselor (Gemini 2.5 Flash)
This is EduFind's most distinctive feature and the primary differentiator from existing platforms like Careers360 and CollegeDunia. Rather than making students manually apply dozens of filters, the AI Counselor has a conversation with the student, understands their profile holistically, and produces a reasoned, ranked shortlist — explaining why each college was or wasn't included.
// lib/gemini.ts — context building

export async function buildCounselorContext(profile: StudentProfile) {
  // Fetch relevant colleges from DB
  const colleges = await prisma.college.findMany({
    where: {
      state: profile.preferredStates?.length ? { in: profile.preferredStates } : undefined,
      feesMax: { lte: profile.budgetLpa * 100000 },
      courses: { some: { branch: { in: profile.preferredBranches } } },
    },
    include: {
      courses: { where: { branch: { in: profile.preferredBranches } } },
      placements: { orderBy: { year: "desc" }, take: 1 },
    },
    orderBy: { nirfRank: "asc" },
    take: 20,
  });

  return colleges.map(c => ({
    name: c.name,
    city: c.city,
    state: c.state,
    nirfRank: c.nirfRank,
    feesPerYear: c.feesMin,
    avgPackageLpa: c.avgPackageLpa,
    jeeMainCutoff: c.courses[0]?.cutoffJeeMain,
    jeeAdvCutoff: c.courses[0]?.cutoffJeeAdv,
    naacGrade: c.naacGrade,
    slug: c.slug,
  }));
}
// app/api/ai/counsel/route.ts

export async function POST(req: Request) {
  const { messages, profile } = await req.json();

  // Build real-data context
  const collegeContext = await buildCounselorContext(profile);

  const systemPrompt = `
    You are EduFind's AI College Counselor for Indian students.
    You have access to real college data from our database.
    Always base recommendations on the DATA provided — never hallucinate.
    
    STUDENT PROFILE:
    ${JSON.stringify(profile, null, 2)}
    
    AVAILABLE COLLEGES (filtered for this student):
    ${JSON.stringify(collegeContext, null, 2)}
    
    Rules:
    - Recommend only colleges from the data above
    - Explain cutoff safety (safe / borderline / risky) for each
    - Format recommendations as a ranked numbered list
    - Be concise but thorough; use a friendly, helpful tone
  `;

  // Stream response from Gemini
  const stream = await geminiClient.streamGenerateContent({
    model: "gemini-2.5-flash",
    contents: [{ role: "user", parts: [{ text: systemPrompt }] }, ...messages],
  });

  // Return as ReadableStream for SSE
  return new Response(stream.body, {
    headers: { "Content-Type": "text/event-stream", "Cache-Control": "no-cache" },
  });
}

Streaming (SSE) | Users see text appear instantly vs. waiting 5–10s for a full response

Conversation history | Full multi-turn context sent on every message for coherent follow-ups

10. Edge Cases & Tradeoffs
Scenario | Handling | Where
User saves same college twice | @@unique([userId, collegeId]) in DB; API returns 409 Conflict | Prisma schema + API route
AI returns college not in DB | System prompt strictly restricts to injected context; output is validated | Gemini prompt engineering
Filter produces 0 results | Empty state component with "Clear filters" CTA shown | CollegeListingPage
Refresh token expired/stolen | Token deleted from DB on use (rotation); old token invalid immediately | Auth refresh route
Compare with 2 vs 3 colleges | UI supports 2-3; comparison rows show "—" for missing data | ComparePanel
College deleted while saved | SavedItem cascades delete (onDelete: Cascade in schema) | Prisma schema
User submits 2 reviews | @@unique([userId, collegeId]) enforced at DB level; API returns 409 | Review model + route
Gemini API down | Counselor shows error toast; degraded mode falls back to filter-based suggestions | CounselorChat error boundary
Very long college name | CSS text-overflow: ellipsis + tooltip on hover in CollegeCard | CollegeCard component
Mobile compare (3 cols) | Compare switches to stacked card layout below md breakpoint | CompareGrid responsive
Decision | Rationale
Cursor pagination over offset | More complex to implement but O(1) performance at any page depth. Offset breaks at 10k+ rows.
Denormalized avg package on College | Avoids JOIN on every listing query. Updated via a background recalculation when new PlacementStat is inserted.
JWT in memory, not localStorage | XSS cannot steal the access token. Slightly worse DX (token lost on refresh) mitigated by httpOnly refresh cookie auto-renewing.
RAG-lite over full vector search | Vector search requires a separate embedding pipeline and pgvector. RAG-lite with structured SQL context is 90% as effective for this use case with far less infrastructure.
ISR over full SSG | Full SSG would require rebuilding on every college data change. ISR gives static performance with 60-second freshness automatically.
Zustand for compare state | Redux is overkill. Zustand + localStorage gives persistence with minimal boilerplate. The compare state is simple enough to not need a reducer pattern.

// lib/validations.ts

export const collegeQuerySchema = z.object({
  q:        z.string().max(100).optional(),
  state:    z.array(z.string().max(50)).max(10).optional(),
  type:     z.array(z.enum(["GOVERNMENT", "PRIVATE", "DEEMED"])).optional(),
  feesMin:  z.coerce.number().min(0).max(10_000_000).optional(),
  feesMax:  z.coerce.number().min(0).max(10_000_000).optional(),
  sort:     z.enum(["nirf_rank","rating","fees_asc","fees_desc","package"]).optional(),
  cursor:   z.string().cuid().optional(),
  limit:    z.coerce.number().min(1).max(48).default(12),
}).refine(d => !d.feesMin || !d.feesMax || d.feesMin <= d.feesMax, {
  message: "feesMin must be <= feesMax",
});

export const reviewCreateSchema = z.object({
  rating: z.number().int().min(1).max(5),
  title:  z.string().min(5).max(100),
  body:   z.string().min(20).max(2000),
  pros:   z.string().max(500).optional(),
  cons:   z.string().max(500).optional(),
  batch:  z.number().int().min(1950).max(new Date().getFullYear()).optional(),
});

Zod validation on all inputs
Optimistic UI for saves
